DASHBOARD EXAMPLES - HOW BOT HANDLES DIFFERENT TYPES

=======================================================================
EXAMPLE 1: E-COMMERCE SALES DASHBOARD
=======================================================================

Dashboard Purpose: Track online sales, customer sentiment, and regional performance

Columns Available:
- sentiment (Positive, Negative, Neutral)
- country (USA, Germany, India, Brazil)
- region (North, South, East, West)
- product_category (Electronics, Home, Clothing)
- order_date
- revenue
- quantity

User Query: "Show me positive sentiment from USA last quarter"

Bot Captures:
{
  "sentiment": "Positive",
  "country": "USA",
  "time_period": "last_quarter"
}

Filters Applied:
[
  {column: "sentiment", operator: "In", values: ["Positive"]},
  {column: "country", operator: "In", values: ["USA"]},
  {column: "order_date", operator: "Between", values: ["2026-04-01", "2026-06-30"]}
]

Result: Dashboard shows only USA orders with positive feedback from Q2 2026

KEY POINTS FOR YOUR LEAD:

1. Same Bot, Different Domains
   - Same code works for retail, HR, marketing, finance, healthcare, education, etc.

2. Generic Processing
   - Bot doesn't know column names in advance
   - Learns from schema registration
   - Adapts to ANY structure

3. Query Understanding
   - "Show X from Y" → filters applied automatically
   - Time periods handled generically (last_month, last_quarter, etc.)
   - Multiple filters combined in one query

4. Scalability
   - Add new dashboard = just register new schema
   - No code changes needed
   - Same pipeline processes all requests

5. Flexibility
   - Numerical ranges (high, low, specific values)
   - Categorical filtering (exact matches)
   - Date range filtering (time periods)
   - Combined filters (multiple conditions)

6. Real-World Use Cases
   - Executives query sales dashboards
   - HR managers query employee data
   - Marketers analyze campaign performance
   - Finance teams track transactions
   - Hospital staff query patient data
   - Teachers check student performance
   - Support teams manage tickets

Each dashboard has different columns but uses the SAME backend, SAME pipeline, SAME bot.
